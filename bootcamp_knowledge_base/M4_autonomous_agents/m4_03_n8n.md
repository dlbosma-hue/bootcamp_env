# LAB M4.03 - n8n Workflow Automation

## What is n8n?
n8n is a visual workflow automation tool. You connect apps and services by dragging and dropping nodes onto a canvas, without writing backend code. It handles the "plumbing" between systems.

Think of it as a visual programming tool where each step in your process is a box, and you draw arrows between boxes to define the flow.

## What n8n Replaces
Without n8n you would need to manually code:
- A Flask or FastAPI web server to receive webhooks
- Cron jobs for scheduling
- Email sending libraries
- Retry logic and error handling
- Authentication flows for each API

n8n gives you all of this through drag-and-drop nodes.

## Key Lab: Discord + Google Sheets / Airtable Integration

### Use Case
Automatically log every Discord message from clients into a Google Sheets or Airtable tracker. No manual copy-pasting.

### The 3-Node Workflow
```
Discord Trigger (or Manual Trigger)
    |
    v
Set Node  <-- transforms Discord data format into Sheets format
    |
    v
Google Sheets / Airtable  <-- appends a new row
```

### The Set Node
The Set node is where data transformation happens. It maps fields from the incoming Discord format to the columns in your spreadsheet:
```
Discord field "content"        -->  Sheets column "Question"
Discord field "author.username" -->  Sheets column "Client Name"
Discord field "timestamp"       -->  Sheets column "Timestamp"
.toISO()  <-- converts Discord timestamp format to ISO 8601 format that Sheets accepts
```

## The 14 Nodes Documented (n8n Node Reference Lab)

### Trigger Nodes
- Chat Trigger - listens for chat messages; starts the workflow when a message arrives
- Manual Trigger - for testing; lets you run the workflow manually with a button click
- Webhook - listens for HTTP POST requests from external systems

### Processing Nodes
- AI Agent - the main AI reasoning node; connects to LLM and tools
- Set - transforms and maps data fields between formats
- Code - runs custom JavaScript or Python logic inline in the workflow

### AI / Memory Nodes
- OpenAI Chat Model - configures which OpenAI model the agent uses
- Simple Memory - stores conversation history linked by session ID; the session ID is what connects the Chat Trigger to the Memory node
- Pinecone Vector Store - connects to your Pinecone index for RAG retrieval
- Embeddings OpenAI - generates embeddings for the text being stored or queried

### Document Nodes
- Default Data Loader - loads a document (PDF, text) into the workflow
- Recursive Character Text Splitter - splits documents into chunks with overlap
- Cohere Rerank - reranks retrieved chunks by relevance score

### Utility Nodes
- HTTP Request - makes API calls to any external service

## n8n vs LangGraph: When to Use Which
- n8n: when you need to connect apps, handle scheduling, receive webhooks, send emails, or route data between systems. Best for "plumbing" and integration.
- LangGraph: when you need the AI itself to make structured, multi-step decisions with conditional logic. Best for the AI reasoning layer.
- They work together: n8n triggers the LangGraph agent, receives results, and routes them to the right destination.

## Key Learning
The session ID is the invisible thread that connects a user's messages across multiple turns. In n8n, the Chat Trigger creates a session ID, and the Simple Memory node stores conversation history indexed by that same session ID. Without matching session IDs, each message appears to be a new conversation with no memory of previous turns.

## Idempotency Note
Running the same n8n workflow multiple times produces duplicate rows in Google Sheets/Airtable because Append/Create operations do not check for duplicates. In production, add a deduplication check before the append step.

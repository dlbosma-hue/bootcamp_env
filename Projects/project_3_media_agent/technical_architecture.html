Local Server — FastAPI on localhost:8000 | Exposed via ngrok tunnel
STARTUP
python run.py
One command starts the server. N8N runs in the cloud (app.n8n.cloud). FastAPI runs locally. Ngrok creates a secure HTTPS tunnel so cloud N8N can reach localhost:8000.
TUNNEL
ngrok http 8000
Creates HTTPS tunnel from N8N cloud → localhost:8000. Required because N8N cloud cannot reach localhost directly. URL changes on restart — update N8N each session.
HTTP SERVER
FastAPI /research
Receives POST from N8N at https://{ngrok-id}.ngrok.io/research. Validates input. Calls LangGraph agent. Returns JSON result.
agent_graph.invoke({company, focus})
Python — LangGraph ReAct Agent (5 Nodes) | gpt-4o-mini reasoning + claude-sonnet-4-6 report
NODE 1
PLAN
gpt-4o-mini reads company + focus. Decides what to research. Sets research agenda in state.
NODE 2
RESEARCH
ReAct loop. Agent calls all 4 MCP tools autonomously. Stores results in state.
NODE 3
ANALYSE
gpt-4o-mini structures tool results by 4 angles x 4 communities. Pydantic enforces output format.
NODE 4
SYNTHESISE
gpt-4o-mini scores each community 1-10. Identifies harm flags and strengths. Calculates overall score.
NODE 5
REPORT
4 claude-sonnet-4-6 calls build the full 11-section prose report. Saves to /reports/ folder. Returns report_text.
4 MCP Tools -- LangChain @tool  called from inside RESEARCH node ↑
TOOL 1
NewsAPI
search_news(query). NEWSAPI_KEY in header. Returns up to 10 articles. 3 retries with backoff.
TOOL 2
Guardian API
search_guardian(query). No API key needed -- public content API. Returns articles + byline data from The Guardian.
TOOL 3
Wikipedia API
get_wikipedia(company). No key needed. Returns company background, ownership, history.
TOOL 4 PRIMARY
RSS Feed Analyser
analyse_rss_feed(company). No key needed. Reads the outlet's own feed directly. Bylines, topics, language flags.
RAG SYSTEM -- Pinecone + OpenAI  rag_query() also called from RESEARCH node ↑
STEP 1
OpenAI Embedding
text-embedding-ada-002 turns the query into a 1536-dimension vector. OPENAI_API_KEY required.
STEP 2
Pinecone Search
media-inclusivity-index. 19 vectors. Returns the 3 most similar chunks. PINECONE_API_KEY required.
STEP 3
Benchmark Context
Retrieved chunks (score >0.70 only) added to state. RESEARCH node continues with benchmarks in hand.
REPORT node complete → FastAPI returns JSON: {status, overall_score, summary, report_text, harm_flags, community_scores}
N8N -- Result Handling + Smart Routing
IF NODE
Check If Success
Reads status field from JSON. Routes to Success path or Error path.
SET NODE
Success Output
Formats the result: company, overall_score, report_text, harm_flags, community_scores, triggered_by.
ERROR PATH
Error Output
If status != success. Logs error + suggestion. Returns 500 to caller.
SWITCH NODE
Route by Score
Single dynamic Notion node. Status = overall_score >= 7 ? 'Good Standing' : >= 4 ? 'Needs Review' : 'Flagged for Action'. 9 blocks support reports up to ~18k chars.
NOTION
Save to Notion
Creates page titled "[Company] | Week N YYYY". Routes to correct database based on score band.
NOTION FLAGGED
Flagged Database
Separate Notion DB for scores below 4. Live watchlist of outlets with serious inclusivity concerns.
SLACK
Slack Alert
Score below 4 only. Auto-sends DM: company name, score, and top harm flags to editorial team.
